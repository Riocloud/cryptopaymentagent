# Global Preferences

## Communication
- Be direct and concise. Assume senior-level engineering knowledge.
- When unsure, say so. Do not guess.
- If a task is ambiguous, ask one clarifying question before proceeding.
- Bilingual: match the language I use in my prompt (English or Chinese).

## Coding
- Prefer simple, readable solutions over clever ones.
- Always verify changes compile/run before marking done.
- Run existing tests after making changes. Do not skip this.
- Conventional commits (feat:, fix:, chore:, etc.)

## Workflow
- State your plan in 2-3 sentences before starting work.
- Small, incremental changes over large rewrites.
- When debugging, check logs and error messages first.

## Environment
- Linux (Ubuntu/Debian) and macOS. Heavy Docker usage.
- Use whatever package manager the project already uses.

## Donts
- Do not add comments that restate the code.
- Do not refactor unrelated code while fixing a bug.
- Do not create new files without telling me first.
